# Sprint 1: Definição do problema
**Imersão, definição, ideação**

Esta é uma atividade prática em grupo que estimula os alunos a especificarem os requisitos do projeto de desenvolvimento de um sistema.

## Problema
Esta tarefa consiste na especificação dos requisitos do projeto para desenvolvimento de um sistema, sendo que espera-se que as seguintes questões sejam respondidas:

* Qual é o tema e o nome do projeto?
* Qual é o problema tratado pelo projeto?
* Para qual público o projeto se destina?
* Quais foram as principais motivações para o desenvolvimento do projeto?
* Qual é o objetivo do projeto?
* Que impacto o projeto trará para seu público e para a sociedade?
* Quais os requisitos funcionais, não funcionais e restrições do projeto?
* Quais as personas e histórias de usuário?

## Produto
### Entrega dos artefatos da Sprint 1:

#### Repositório no GitHub:
* Preenchimento do **README**
* Preenchimento da **Documentação de Contexto** (item 1)
* Preenchimento da **Especificação do Projeto** (item 2)
* Atualizar **GitHub Projects**
* Preenchimento das **Referências**
* Inserir a URL do repositório Git nos slides e nos comentários de envio da tarefa

#### Planejamento do Projeto (também no GitHub):
Na documentação postada no GitHub, em **Planejamento do Projeto** (item 3), cada integrante do grupo deverá informar as ações efetuadas em cada Sprint. Veja as subseções "Divisão de Papéis" e "Quadro de tarefas".
Cada integrante do grupo deverá fazer um comentário de envio na tarefa no Canvas explicitando as suas contribuições na Sprint vigente.

## Apresentação do projeto
Os grupos deverão entregar uma apresentação (conjunto de slides). Os grupos também deverão realizar uma apresentação oral de seus resultados em **no máximo 5 (cinco) minutos**. Todos os integrantes do grupo devem apresentar. 

> **Importante:** o grupo deve providenciar laptop e adaptador para apresentação (dia 18/03/2026).

## Resumo das Entregas
* Arquivo **.zip** do repositório GitHub + link do repositório nos comentários de envio.
* Arquivo com a apresentação de slides.
* Comentário de envio de cada integrante no Canvas explicitando as suas contribuições na Sprint vigente.
